Upute korištenja

Funkcionalnost se može provjeriti direktnim postavljanjem zahtjeva, na primjer u Postman aplikaciji.

Lista akcija sa pridruženim adresama:

  1. Otvaranje accounta: POST zahtjev na /account sa tijelom(JSON) sljedećeg oblika. { accountId : "myAccountId" }
  2. Prikaz svih računa: GET zahtjev na /all. Ovoga ne bi trebalo biti ovdje, ali sam ostavio zbog lakše provjere.
  3. Registracija URL-a: POST zahtjev na /register sa tijelom(JSON) sljedećeg oblika. { url : "www.someurl.com", redirectType : 302 } Polje "redirectType" nije obavezno. Za registraciju je potrebna autentifikacija, to se u Postmanu može postaviti kao header 'BasicAuth' s imenom i lozinkom stvorenog računa.
  4. Dohvat statistike: GET zahtjev na /statistic/{accountId} Za registraciju je potrebna autentifikacija, to se u Postmanu može postaviti kao header 'BasicAuth' s imenom i lozinkom stvorenog računa.
  5. Redirekcija: GET zahtjev na /{accountId}/{shortUrl} Jednom kad je stvoren skraćeni URL, stvarnom URL-u se može pristupiti preko navedenog URI-a, gdje je accountId, ID onog tko je stvorio skraćeni URL, a shortUrl generirani skraćeni URL.
